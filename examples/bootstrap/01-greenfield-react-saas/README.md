# Bootstrap Example 01 — Greenfield React SaaS

A new multi-tenant B2B SaaS app built on React + Tailwind + Radix
primitives. The product supports per-tenant branding (white-label
accent color, optional logo) and dark/light mode. The team has no
prior DESIGN.md, no design contract, and no normative domain.

This example shows BOOTSTRAP at its most common: a fresh project that
benefits from `themingAxes`, `tokenTiers`, `runtime`, and an
`antiPatterns` catalog, but has no invariants because the domain
("none/other") imposes none.

## Files

- `input.md` — captures the project state BOOTSTRAP would detect plus
  the user's answers to the eight interview questions.
- `expected-output/DESIGN.md` — the DESIGN.md the skill should
  produce. Used by the test suite as a golden file.
- `notes.md` — what this example demonstrates and what it
  intentionally does NOT cover.
